# Как сделать из скринкаста красивые обрезанные изображения для user manual

## Что получится

На выходе вы получите набор картинок, которые можно вставлять в инструкцию:

- `shot_01.jpg`
- `shot_02.jpg`
- `shot_03.jpg`

Каждая картинка будет:

- извлечена из видео по таймкоду;
- обрезана до нужной области;
- подготовлена для вставки в manual.

При необходимости тот же процесс можно использовать и для `PNG`.

## Что понадобится

- Linux / Ubuntu
- видеофайл, например `video.mp4`
- список таймкодов
- `FFmpeg`
- `GIMP` для выбора области обрезки

## Когда выбирать JPEG, а когда PNG

### Используйте JPEG, если:

- важен небольшой размер файлов;
- изображений много;
- небольшая потеря качества допустима.

### Используйте PNG, если:

- в кадре много текста;
- это интерфейс системы;
- нужна максимальная четкость.

### Практическое правило

Для user manual с интерфейсом чаще лучше `PNG`, но если по задаче нужны именно `JPEG`, используйте их с высоким качеством через `-q:v 2`.

## Шаг 1. Установить FFmpeg

Откройте терминал:

```bash
Ctrl + Alt + T
```

Проверьте, установлен ли `FFmpeg`:

```bash
ffmpeg -version
```

Если команда не найдена, установите программу:

```bash
sudo apt update
sudo apt install ffmpeg
```

После этого снова проверьте:

```bash
ffmpeg -version
```

## Шаг 2. Перейти в папку проекта

Перейдите в папку, где лежат ваши файлы:

```bash
cd "/путь/к/проекту"
```

Пример:

```bash
cd "/media/ksenia/720CA17A0CA13A4D/WORK/Gen1POS"
```

Проверьте содержимое:

```bash
ls
ls docs
```

## Шаг 3. Создать папки для работы

Создайте две папки:

- `docs/screens_full` — для исходных кадров;
- `docs/screens_cropped` — для итоговых обрезанных изображений.

```bash
mkdir -p docs/screens_full
mkdir -p docs/screens_cropped
```

## Шаг 4. Подготовить таймкоды

Для каждого шота нужен один момент времени, в который будет снят кадр.

Пример:

```text
01,00:00:01
02,00:00:06.5
03,00:00:17
04,00:00:31
```

### Как выбирать момент кадра

Лучше брать:

- не самое начало шота;
- а середину или удачный момент внутри шота.

Так меньше шанс получить:

- переход;
- мигание;
- пустой экран;
- неудачное положение курсора.

## Шаг 5. Извлечь кадры из видео

### Вариант A. По одной команде

Если кадров немного, можно запускать по одному.

```bash
ffmpeg -y -ss 00:00:01 -i "docs/video.mp4" -frames:v 1 -q:v 2 "docs/screens_full/shot_01.jpg"
ffmpeg -y -ss 00:00:06.5 -i "docs/video.mp4" -frames:v 1 -q:v 2 "docs/screens_full/shot_02.jpg"
ffmpeg -y -ss 00:00:17 -i "docs/video.mp4" -frames:v 1 -q:v 2 "docs/screens_full/shot_03.jpg"
```

### Вариант B. Серией

Если кадров много, используйте цикл:

```bash
while IFS=, read -r num ts; do
  ffmpeg -y -ss "$ts" -i "docs/video.mp4" -frames:v 1 -q:v 2 "docs/screens_full/shot_${num}.jpg"
done <<'EOF'
01,00:00:01
02,00:00:06.5
03,00:00:17
04,00:00:31
EOF
```

### Что означают параметры

- `-ss` — перейти к нужному времени;
- `-i` — входной видеофайл;
- `-frames:v 1` — сохранить один кадр;
- `-q:v 2` — высокое качество JPEG;
- `shot_01.jpg` — имя итогового файла.

## Шаг 7. Выбрать область, которую нужно оставить

Исходные кадры из скринкаста часто содержат лишнее:

- рабочий стол;
- системную панель;
- черные поля;
- лишние элементы по краям.

Чтобы сделать аккуратные иллюстрации, нужно один раз определить область обрезки.

### Как сделать это в GIMP

1. Откройте любой кадр из `docs/screens_full`.
2. Выберите инструмент **Кадрирование**.
3. Выделите область, которую хотите оставить.
4. Посмотрите значения:
   - `Позиция X`
   - `Позиция Y`
   - `Размер X`
   - `Размер Y`

## Шаг 8. Перевести параметры GIMP в crop для FFmpeg

В `GIMP`:

- `Позиция X` = `x`
- `Позиция Y` = `y`
- `Размер X` = `width`
- `Размер Y` = `height`

В `FFmpeg` это записывается так:

```bash
crop=width:height:x:y
```

### Пример

Если `GIMP` показывает:

- `Позиция X = 3`
- `Позиция Y = 117`
- `Размер X = 1664`
- `Размер Y = 887`

то параметр будет таким:

```bash
crop=1664:887:3:117
```

## Шаг 9. Проверить обрезку на одном файле

Сначала протестируйте обрезку на одном кадре:

```bash
ffmpeg -y -i "docs/screens_full/shot_01.jpg" -vf "crop=1664:887:3:117" -q:v 2 "docs/screens_cropped/test.jpg"
```

Откройте `test.jpg` и проверьте, что:

- лишний фон исчез;
- нужная часть интерфейса осталась;
- картинка выглядит аккуратно.

Если нужно, скорректируйте выделение в `GIMP` и повторите тест.

## Шаг 10. Применить crop ко всем изображениям

Когда тестовый результат вас устраивает, обрежьте все файлы:

```bash
for f in docs/screens_full/*.jpg; do
  name=$(basename "$f")
  ffmpeg -y -i "$f" -vf "crop=1664:887:3:117" -q:v 2 "docs/screens_cropped/$name"
done
```

## Шаг 11. Проверить итог

Проверьте количество итоговых файлов:

```bash
ls docs/screens_cropped | wc -l
```

Посмотрите список:

```bash
ls docs/screens_cropped
```

## Шаг 12. Отобрать лучшие изображения для manual

Не обязательно использовать все кадры подряд.

Оставляйте те, которые:

- показывают конкретный шаг;
- хорошо читаются;
- не повторяют соседние почти полностью;
- полезны для объяснения процесса.

## Готовый шаблон под новый проект

### Подготовка

```bash
cd "/путь/к/проекту"
mkdir -p docs/screens_full
mkdir -p docs/screens_cropped
```

### Извлечение кадров в JPEG

```bash
while IFS=, read -r num ts; do
  ffmpeg -y -ss "$ts" -i "docs/video.mp4" -frames:v 1 -q:v 2 "docs/screens_full/shot_${num}.jpg"

01,00:00:01
02,00:00:06
03,00:00:15
EOF
```

### Тестовая обрезка

```bash
ffmpeg -y -i "docs/screens_full/shot_01.jpg" -vf "crop=WIDTH:HEIGHT:X:Y" -q:v 2 "docs/screens_cropped/test.jpg"
```

### Обрезка всех файлов

```bash
for f in docs/screens_full/*.jpg; do
  name=$(basename "$f")
  ffmpeg -y -i "$f" -vf "crop=WIDTH:HEIGHT:X:Y" -q:v 2 "docs/screens_cropped/$name"
done
```

## Когда лучше использовать PNG

Используйте PNG, если:

- в интерфейсе много мелкого текста;
- нужна максимальная четкость;
- размер файлов не критичен.

Тогда:

- замените `.jpg` на `.png`;
- уберите `-q:v 2`.

Пример:

```bash
ffmpeg -y -ss 00:00:17 -i "docs/video.mp4" -frames:v 1 "docs/screens_full/shot_01.png"
```
